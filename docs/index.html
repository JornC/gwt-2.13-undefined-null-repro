No configuration, no reflection — just a field read, an autobox, and a null check. Only the GWT version differs between the two boxes above.

Root cause: GWT PR #10165 (commit b5155aa) made JsUtils.uncheckedCast inlinable. The autobox double→Double runs through it, so after inlining the optimizer proves the boxed value is non-null (it came from a primitive) and constant-folds boxed == null to false. The undefined that arrived from JS is never normalized to null. Only the optimized/production compile is affected; SuperDevMode is not.

Fix in your code: declare the native-overlay field as the boxed type (Double, not double) so it is read as a nullable reference.

Proposed compiler fix (third panel above): a one-line guard in JUnsafeTypeCoercion.getType() that stops strengthening a coerced primitive to non-null, so undefined-bearing values stay nullable — while keeping #10165's inlining. That panel is the repro compiled with a gwt-dev-2.13.1 jar carrying only this patch; it answers true again.

Reproduce locally: ./run.sh compiles both versions and prints the runtime and constant-fold signals. Source and README on GitHub.
